# 🧠 PRODUCT ONE-PAGER

## Wallet Companion (Elderly Mode inside eWallet)

# 📌 EXECUTIVE SUMMARY

Malaysia’s elderly face a growing financial independence gap — not because financial tools
don’t exist, but because they are **too complex, intimidating, and not designed for daily use**.
This product introduces a **simplified “Elderly Mode” inside the existing eWallet** , transforming
a complex financial interface into a **voice-first, low-friction companion** that helps seniors:
● Understand their money daily
● Make simple financial decisions
● Build confidence over time
👉 Instead of adding new features, we **remove complexity and redesign behaviour**

# ❓ HOW MIGHT WE

```
How might we help elderly users confidently manage their daily finances
using tools they already have — without increasing complexity?
```

# 👤 PERSONA & PROBLEM

## 🎯 Primary Persona: Mak Cik Aishah (60–75)

### Profile

```
● Uses smartphone (WhatsApp, basic apps)
● Low financial literacy
```

```
● Irregular / limited income
● Risk-averse, avoids making mistakes
```

### Core Problems

```
● ❌ Doesn’t know if her money is enough daily
● ❌ Finds financial apps confusing and overwhelming
● ❌ Afraid of making wrong financial decisions
● ❌ Vulnerable to scams and poor judgement
● ❌ No habit of tracking or reflecting on spending
```

# 🧭 DESIGN PRINCIPLES

1. **Clarity over completeness**
   → Show only what matters _today_
2. **Conversation over navigation**
   → Let users ask instead of explore
3. **Guidance over data**
   → Translate numbers into decisions
4. **Confidence over control**
   → Reduce fear before adding features
5. **Habit over one-time usage**
   → Build daily engagement loops

# 💡 SOLUTION (ONE-LINER)

```
A simplified, voice-first financial companion embedded within the eWallet that helps
elderly users understand, decide, and build confidence in managing their
money daily
```

# 🧩 MODULES & FEATURES (MVP SCOPE)

## 1 ⃣ Elderly Mode (Entry Layer)

**Purpose:** Reduce complexity instantly

### Features:

```
● Entry tile from main eWallet (“Mode Mudah”)
● Simplified UI (large buttons, minimal actions)
● Exit back to normal mode
```

## 2 ⃣ Voice Companion (“Ask Me”)

**Purpose:** Enable decision-making without navigation

### Features:

```
● Voice input (Malay-first)
● Predefined intents:
○ Balance check
○ Spending summary
○ Affordability (“Boleh beli ini?”)
● Simple, human-readable responses
● Optional text-to-speech output
```

## 3 ⃣ Daily Check-In (Engagement Layer)

**Purpose:** Build habit and awareness

### Features:

```
● Daily spending summary
● Budget status indicator (Safe / Warning / Exceeded)
● Positive reinforcement messages
● Passive visibility on home screen
```

## 4 ⃣ Wallet Simulation (Core Logic)

**Purpose:** Provide a simple financial state

### Features:

```
● Daily budget setup (preset options)
● Balance tracking
● Basic transaction logging (manual input)
● Rule engine:
○ Compare spend vs budget
○ Trigger feedback
```

## 5 ⃣ Add Spending (Action Layer)

**Purpose:** Enable simple behaviour tracking

### Features:

```
● Quick RM input (numeric keypad)
● One-tap save
● Instant balance update
```

## 6 ⃣ Safety Nudges (Trust Layer)

**Purpose:** Prevent risky actions and build trust

### Features:

```
● Large amount detection
● Spending warnings (low balance / overspending)
● Simple confirmation prompts
● Scam awareness messaging
```

# 🎯 MVP FOCUS (24-HOUR BUILD)

**Must Have:**

● Elderly Mode UI
● Voice / button-based queries
● Daily summary + budget logic
● Add spending
**Nice to Have:**
● Voice output
● Safety nudges

```
We are not building a finance tool —
```
